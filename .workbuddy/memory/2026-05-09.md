## 修复 Eventbrite 活动重复问题

### 问题
用户反馈 BuffBoyzz Gay Friendly Male Strippers 的活动总是出现两条重复的。

### 根因
`server/api/events/eventbrite.ts` 中使用 `formatTitleAndDateToID()` 函数生成活动 ID，该函数基于：
- 日期时间 (YYMMDDHHMM)
- 标题前 3 个字母

**问题**：如果同一组织者在同天同时段举办多个活动，且标题前 3 个字母相同，就会生成相同的 ID，导致重复或覆盖。

### 修复内容 (`server/api/events/eventbrite.ts`)
- **删除** `formatTitleAndDateToID()` 函数（不再使用）
- **改用 Eventbrite 原生 ID**：`id: \`eventbrite-${item.id}\``
  - Eventbrite API 返回的每个活动都有唯一的 `item.id`
  - 使用原生 ID 彻底避免冲突

### 技术细节
```typescript
// 修改前（有冲突风险）
id: formatTitleAndDateToID(startDateObj, title),

// 修改后（使用 Eventbrite 原生唯一 ID）
id: `eventbrite-${item.id}`,
```

## 修复时间提取错误（noon → 08:00 AM 问题）

### 问题
"Pups & Pride" 活动海报写的是中午12点（noon），但被提取成了 08:00 AM。

### 根因
1. `analyzeWithAI()` 的 prompt 对 "noon" 的说明不够突出，AI 容易忽略
2. JS 的 `||` 运算符有 falsy bug：`0 || 12` = `12`，导致午夜（0）被错误替换成中午（12）

### 修复内容 (`scripts/update-calendar.ts`)
1. **强化 prompt**：
   - 将时间解析规则分行写明，每行格式统一：`"noon" or "12pm" or "12:00pm" = 12 (NOT 0, NOT 8, NOT 13)`
   - 在结尾 reminder 中再次强调：`noon/12pm = 12`
2. **修复 `||` → `??`**：
   - `ev.startHourMilitaryTime || 12` → `ev.startHourMilitaryTime ?? 12`
   - `ev.startMinute || 0` → `ev.startMinute ?? 0`
   - `ev.endHourMilitaryTime || 23` → `ev.endHourMilitaryTime ?? 23`
   - `ev.endMinute || 59` → `ev.endMinute ?? 59`
   - `endHourMilitaryTime` 判断：`if (ev.endHourMilitaryTime)` → `if (ev.endHourMilitaryTime !== null && ev.endHourMilitaryTime !== undefined)`
   - `??` 只对 `null/undefined` 生效，`0`（午夜）会正确保留
